Step 1: All-in-One WordPress Plugin

  1. Login to development server site (yourdomain.drnoce.biz/wp-admin).
  2. Once on the Dashboard, locate the grey sidebar on the left and select "Plugins"
  3. Click "Add New" located on either the grey sidebar or at the top of the page.
  4. Search for "All-In-One WP Migration"
  5. Click "Install Now." The plugin will now install.
  6. After installed, click the blue "Activate" button.
  7. Towards the button of the grey sidebar, click "All-in-One WP Migration."
  8. Ensure you are in export mode.
  9. Press the green "Export to" button and select "File."
  10. The site will begin preparing the website to be exported. Note: this process may take a long time. Once completed download the file (file extension: .wpress). Take note of the file size.
  11. Log onto the client's provided WordPress account (This should not end in .drnoce.biz)
  12. If the site export is less than 512 mb, repeat Steps 2-7 to install the "All-In-One WP Migration" Plugin on the client's webspace. If the site export is more than 512 mb, follow these steps instead.
  13. Select import in the grey sidebar and ensure you are in import mode.
  14. Upload the .wpres file. Note: this process may take a long time.
  15. Once completed, click permalink settings in the box.
  16. Click "Save Changes." After this, click "Save Changes" again.

    17. If My Export File is Larger than 512mb:

    The default version of All-In-One WP Migration has an import limit of 512mb. Partnership Erie has a modified program that allows for larger file sizes to be uploaded.

    1. Contact Dr. Noce or visit Teams to find the Modified All-In-One WP Migration .zip.
    2. Once on the Dashboard, locate the grey sidebar on the left and select "Plugins"
    3. Click "Add New" located on either the grey sidebar or at the top of the page.
    4. Towards the top of the site, click "upload plugin" and navigate to the .zip file provided.
    5. Activate the plugin.
    6. Return to Step 13.

Step 2: FTP Backup

Note: It is not required to backup FTP/SFTP before SQL. Both will backup fine regardless of the order.

  1. Log into DreamHost
  2. Once logged in, locate the left side panel and select "FTP & SSH Users"
  3. From the drop down, select "Manage Users"
  4. All FTP accounts will now be displayed.
  5. Locate the FTP user associated to your domain and click "Show Info"
  6. The host, username, and port will be listed. Put this information aside for Step 12. Click "Reset Password"
  7. Input a new password into the text field and click the blue "Reset Password" button
  8. DreamHost may take a few minutes to reset the SFTP password
  9. Once the password is reset, the same panel in FTP users will be diplayed showing the new password. You have successfull changed and acquired the SFTP/FTP password.
  10. Using the host, username, and password, we start setting up to connect via SFTP/FTP

    11. You can use DreamHost's "Manage Files" button in the top right of the FTP/SFTP user or an external FTP client. I reccomend using Filezilla as it is often faster and more reliable than browser transfers.

    You can download Filezilla from their website here

  11. Open FileZilla. The window should lool similar to the image below.
  12. Next, input your host, username, password, and port and click "Quick Connect". This information was collected in Step 9
  13. You may be alerts to an "Unknown Host Key". Click "OK"
  14. Upon successfully connecting, you will see the sever contents appear in the right-side panel.

    15. The left side is your local directory/PC.

  15. Navigate in the left panel to where you would like to store your backup. Then drag and drop your domain from the right panel to the left.
  16. You will see activity in "Queued Files" section at the bottom of FileZilla signifying the transfer of files. This process can take 5-120 minutes to complete depending on the website, computer, and server, so feel free to grab a coffee or something while we wait.
  17. Upon completion, the "Queued Files" section mentioned in the previous step will be empty.

    18. For reference, the basic WordPress installation used in this example transfered 5,911 files in total.

  18. The SFTP/FTP transfer is now complete.

Step 3: SQL Backup

This method uses DreamHost's internal website and phpMyAdmin platform to access and export MySQL databases. - Reference

MySQL databases can also be backed up via SSH - Reference

Backing up the SQL will require a similar process as FTP.

  1. Log into DreamHost
  2. Once logged in, locate the left side panel and select "More"
  3. From the "More" drop down, select "MySQL Databases"
  4. A page will load displaying options to create a new SQL Database and diplsay all current MySQL servers currently hosted in the DreamHost account
  5. Locate your desired domain and copy the hostname (MYSQL.MYDOMAIN.Drnoce.biz) address into a seperate document for later use

    6. Additionally, middle click or right click "phpMyAdmin" and open it as a new window or tab. We will use this page to log in to the database.

  6. Then, continue down the page until you have located your domain under "Databases on this server".

    7. Then click the item in third column, usually YourDomaindrnocebiz or something similar. This text is blue and links to another page. Click on that text.

  7. The MySQL user page will be displayed.

    8. At the top of the page, you will see the header: "User XXXXX has access to the following ....". Copy word/phrase after user aside for later use.

    This is the same text that was clicked on in the previous step.

    This is your MySQL Database username.

  8. Next, scroll down and locate "Current Password". Click the "Show" button to display the user's SQL password. Put this information aside.
  9. Then, return to the tabbed or seperate windowed phpMyAdmin page. It should look similar to the image below.
  10. Input the hostname, username, and password collected from previous steps and click "Go" in the bottom right.
  11. Upon logging in, phpMyAdmin will now be displayed.
  12. In the top naviation menu, click "Export"
  13. In most cases, a "Quick" export method will suffice.

    14. In cases of extra scripting, database magic, or excess database information (such as eCommerce, forums, etc), a "Custom" export may be required.

    See additional information on phpMyAdmin's Docs here

    Once Quick or Custom options are selected, click the "Go" button

  14. Your browser will start the download of the SQL Database.
  15. You have successfully exported and backed-up a MySQL Database!
